# V87 - 钱币数量与拖拽落位修复

本版本基于 V86 继续修改：

1. 修复终局结算中钱币数量过大无法提交的问题。
   - 钱币类物品不再使用 99 的数量上限。
   - 非钱币物品仍保留原有单项 99 的保护上限。

2. 修复物品录入中钱币数量受 20 限制的问题。
   - 钱币类物品录入不设总数量上限。
   - 非钱币物品仍保留单次 20 的保护上限。

3. 钱币堆叠增加单堆上限。
   - 每个钱币堆叠最多 9999。
   - 超过 9999 的数量会自动拆成多个堆叠。
   - 拖拽钱币到同类钱币时，会尽量补满目标堆叠，剩余数量保留在原堆。

4. 仓库拖拽落位优化。
   - 目标位置被占用时，系统会自动寻找最接近的可用空间。
   - 只有没有任何可用空间时才提示失败。

部署时继续保留：

```text
data/app.db
static/uploads/
```
